4 Configuring the client

For a JPPF client to submit a job for remote execution, we need to specifiy where the remote server is located and which communication ports will be used. This is done through a configuration file, which the JPPF client reads and processes when it is initialized. This configuration file follows the specifications for Java properties.

First, we specify how many servers the client connects to, and we name these server connections:

$template{name="code_start"}$jppf.drivers = HelloServer$template{name="code_end"}$

The next step is to specify the host name or IP address of the server

$template{name="code_start"}$HelloServer.jppf.server.host = localhost$template{name="code_end"}$

We then specify the two ports required by the server: one for dynamically downloading client-side classes (e.g. your application classes), and a second port for the submission of the jobs and reception of the results.

$template{name="code_start"}$HelloServer.class.server.port = 11111 HelloServer.app.server.port = 11112$template{name="code_end"}$

Note: these configuration properties, except for jppf.drivers, all have a default value, which we are using. These default values are used when a property is not specified in the configuration, hence specifying them is actually optional here.

To enable local execution of the JPPF jobs:

$template{name="code_start"}$jppf.local.execution.enabled = true$template{name="code_end"}$

Optionally, we can also specify the number of threads allocated to local execution - the default being the number of processors available to the JPPF client's JVM:

$template{name="code_start"}$jppf.local.execution.threads = 2$template{name="code_end"}$

To recapitulate, here is a full configuration file:

$template{name="code_start"}$#------------------------------------------------------------------------------# # List of drivers this client may connect to # #------------------------------------------------------------------------------# jppf.drivers = HelloServer #------------------------------------------------------------------------------# # Host name, or ip address, of the host the JPPF driver is running on # #------------------------------------------------------------------------------# HelloServer.jppf.server.host = localhost #------------------------------------------------------------------------------# # port number for the class server that performs remote class loading # # default value is 11111; uncomment to specify a different value # #------------------------------------------------------------------------------# HelloServer.class.server.port = 11111 #------------------------------------------------------------------------------# # port number the clients / applications connect to # # default value is 11112; uncomment to specify a different value # #------------------------------------------------------------------------------# HelloServer.app.server.port = 11112 #------------------------------------------------------------------------------# # Enable local execution of tasks? Default value is false # #------------------------------------------------------------------------------# jppf.local.execution.enabled = true #------------------------------------------------------------------------------# # Number of threads to use for loacal execution # # The default value is the number of CPUs available to the JVM # #------------------------------------------------------------------------------# jppf.local.execution.threads = 2$template{name="code_end"}$

This configuration file can be downloaded here: jppf-client.properties
