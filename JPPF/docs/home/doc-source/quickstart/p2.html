2 Writing your first JPPF task

2.1 Some definitions

In JPPF terminology, the basic unit of execution is called a task. It is the smallest self-contained piece of code that can be executed remotely.

JPPF is particularly suited for a class of problems called embarassingly parallel problems, where a problem or program can be split into many independent tasks, and each task can be executed in parallel. We will see how to write such tasks with JPPF.

2.2 Writing a "hello world" task

Here is our first JPPF task. The goal of this task is to create a string message "Hello World", display it in the console of the JVM where it is running, and finally set this message as the task's execution result.

$template{name="code_start"}$ package org.jppf.helloworld; import org.jppf.server.protocol.JPPFTask; /** * A simple hello world JPPF task. */ public class HelloWorld extends JPPFTask { /** * Execute the task. * @see java.lang.Runnable#run() */ public void run() { String hello = "Hello, World"; System.out.println(hello); setResult(hello); } }$template{name="code_end"}$

Note: the elements of text in bold in the code above are hyperlinks that will lead you to the Javadoc for the related API.

The first thing we can see is that our class HelloWord extends the JPPF class JPPFTask. This is the base class for all tasks that can be executed with JPPF. We will see later on that, even when reusing existing code that does not use any JPPF API, this code is actually wrapped into a sublcass of JPPFTask.

The JPPFTask class requires that we implement the run() method. This method is inherited from the Java Runnable interface, which JPPFTask implements and adds functionality to.

Finally, we can observe that our code is calling the setResult(hello) method, to set the "Hello World" message as the task's execution result. This result can be later retrieved by calling getResult(), when the task has been returned to the JPPF client that submitted it.
